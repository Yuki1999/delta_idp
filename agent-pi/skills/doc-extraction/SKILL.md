# 报关资料关键信息抽取 Skill

## 使用场景

当用户上传或指定一套报关资料（客户邮件 + 商业发票 + 装箱单 + 业务说明）时，必须使用本 Skill。
支持两种模式：单文件抽取（传统模式）和文件夹抽取（报关资料集模式）。

## 文件夹抽取模式（推荐）

### Step 1. 接收综合文档内容

系统已将文件夹内所有文件解析并合并为一个文档，包含：
- 客户委托邮件（.txt）
- 业务说明（.txt）
- 商业发票 Invoice（.xlsx → MinerU 解析）
- 装箱单 Packing List（.xlsx → MinerU 解析）

每个文件以 `## 文件：文件名` 标记分隔。

### Step 2. 识别各文件类型

根据文件名和内容判断：
- 客户邮件.txt → 包含委托信息、业务类型、口岸、联系人
- 业务说明.txt → 包含报关单位、运输方式、监管方式
- *_IV*.xlsx → 商业发票（金额、币制、单价、总价）
- *_PL*.xlsx → 装箱单（箱号、毛重、净重、体积、件数）

### Step 3. 交叉引用填写22个报关字段

| 序号 | 字段 | 数据来源 |
|------|------|----------|
| 1 | 委托单位 | 客户邮件/资料客户信息 |
| 2 | 报关单位 | 业务说明中指定的供应商 |
| 3 | 业务类型 | 客户邮件内容（进料加工/来料加工/一般贸易等） |
| 4 | 发货单位 | 客户资料+业务类型 |
| 5 | 收货单位 | 客户资料+业务类型 |
| 6 | 运输方式 | 业务类型+口岸信息 |
| 7 | 件数 | 装箱单 |
| 8 | 毛重 | 装箱单 |
| 9 | 净重 | 装箱单 |
| 10 | 体积 | 装箱单 |
| 11 | 包装方式 | 装箱单 |
| 12 | 发票号码 | 发票 |
| 13 | 目的地/装货港 | 业务类型（出口填目的地，进口填装货港） |
| 14 | 进出境关别 | 客户邮件 |
| 15 | 起抵运地 | 发票/业务说明 |
| 16 | 产品编号 | 发票(优先)/装箱单(校验补充) |
| 17 | 申报数量 | 发票(优先)/装箱单(校验补充) |
| 18 | 币制 | 发票 |
| 19 | 总价 | 发票 |
| 20 | 原产国 | 发票/装箱单 |
| 21 | 最终目的国 | 业务类型+起抵运地 |
| 22 | 法定数量 | 商品编码系统（需人工确认） |

### Step 4. 抽取商品明细

**以发票(IV)为主数据源**，保留发票中每一行明细的所有属性列。
装箱单(PL)仅用于校验和补充：
- 校验发票中的数量、件数等是否一致
- 补充发票中缺失的信息（如箱号、毛重等）
- 发票与装箱单不一致时，以发票为准并备注差异

明细列至少包含：序号 | 产品编号 | 品名 | 申报数量 | 单位 | 单价 | 金额 | 原产国
- **绝对禁止省略、合并、截断或使用 “...” 代替**
- 最后加合计行

### Step 5. 输出

Markdown 表格输出：
1. 先输出「报关单主要信息」表格（字段|值|置信度|数据来源）
2. 再输出「商品明细」表格
3. 重量保留3位小数，体积保留4位小数，金额保留2位小数
4. 缺失字段填 null，不得编造
5. 置信度标注：高/中/低

## 单文件抽取模式（传统）

### Step 1. 读取文件

如果用户提供文件路径或文件名，必须先调用 `read_document` 工具。

该工具会调用 `scripts/read_document.py` 完整解析 xlsx 文件，输出包含所有行、所有单元格的结构化文本。ITEM 明细行会逐行标记为 `ITEM_DATA`，保证一行不漏。

**禁止在没有读取文件内容的情况下直接抽取。**

### Step 2. 获取模板

调用 `list_extraction_templates` 工具，获取系统可用模板。

### Step 3. 匹配模板

根据文档内容匹配：
- 含金额/币制/贸易条款/单价/总价 → invoice
- 含箱号/毛重/净重/体积/包装方式 → packing_list
- 单据号 DS/ES 开头或发货方含 SAMSUNG → samsung
- 其他 → generic

### Step 4. 抽取主字段

按模板字段分组抽取。缺失字段填 null，不得编造。

### Step 5. 抽取 ITEM 明细

从 read_document 输出的 `ITEM_DATA` 行中提取所有明细。

**关键规则**：
- 文档有多少行 ITEM_DATA，输出就必须有多少行
- **绝对禁止省略、合并、截断或使用 "..." 代替**
- 最后加合计行

### Step 6. 输出

Markdown 表格，按分组输出。重量 3 位小数，体积 4 位，金额 2 位。缺失字段为 null。置信度：高/中/低。

## 可用工具

- `read_document` — 调用 `scripts/read_document.py`，完整解析 xlsx，输出含 ITEM_DATA 标记的结构化文本
- `list_extraction_templates` — 列出系统可用模板
